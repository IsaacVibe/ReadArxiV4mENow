# Arxiv Astro-ph Solo Agent Spec

## Why
用户希望直接利用当前环境（Solo/Trae AI助手）的能力来完成天体物理（astro-ph）最新论文的读取、汇总与交互式检索。由于独立的可交互 HTML 页面无法直接在纯 AI 引擎内部运行后端逻辑，因此选择最符合“一切都在 Solo 内进行”需求的形式：**将当前的对话交互界面作为最终的可交互载体**。无需用户配置环境或运行任何外部代码。

## What Changes
- **免代码实现**：不生成、不运行外部 Python 爬虫或前端代码项目。
- **内置工具抓取**：由 Solo 助手（即我本人）使用内置的 `WebFetch` 或搜索工具，实时读取 `https://arxiv.org/list/astro-ph/new` 的内容。
- **对话内汇总输出**：助手在对话窗口中以结构化、美观的 Markdown 格式输出今日文章总数以及各子领域的文章数量分布。
- **对话内交互式检索**：汇总报告生成后，助手将进入等待状态；用户可直接在当前对话中输入感兴趣的领域或关键词（如 Fast Radio Burst, FRB），助手将基于已读取到的上下文，进行精准的语义和模糊检索，并回复相关论文信息。

## Impact
- Affected specs: 方案从“编写独立脚本/页面”转变为“完全基于当前 AI 助手的工具链和对话上下文完成任务”。
- Affected code: 无需生成业务代码。仅通过任务流引导助手完成抓取与对话交互。

## ADDED Requirements
### Requirement: 助手实时读取与汇总
助手需要能使用工具抓取目标网页，并准确理解网页内容。

#### Scenario: 成功抓取与汇总
- **WHEN** 计划获批，开始执行任务
- **THEN** 助手成功读取网页内容，并在对话框中输出文章总数和领域分类统计。

### Requirement: 基于上下文的自然语言交互检索
助手能够记住已读取的论文列表，并在后续对话中响应用户的检索请求。

#### Scenario: 成功检索
- **WHEN** 汇总输出后，用户在对话框发送如 "FRB" 的关键词
- **THEN** 助手从抓取到的上下文中检索出相关论文（包含 arXiv 号、标题、作者），并在对话框中回复。
